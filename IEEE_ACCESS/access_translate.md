# 背景優勢条件下における中距離樹種分類のための深度正則化スパース Multiple Instance Learning

著者: Takahiro Matsumura

## 要旨

地上レベル画像からの樹種分類は、都市林管理、果樹園管理、生態モニタリングにおいて重要である。既存の認識システムは、葉、花、果実、樹皮の近接画像を前提とすることが多い。一方、実際のフィールド画像は中距離から撮影されることが多い。この設定では、幹、枝、樹皮から得られる識別的な手掛かりは画像の一部だけを占め、建物、道路、空、標識、車両、隣接する樹木が背景の大部分を占める。そのため、画像レベルのラベルは利用できるが、有用な領域は疎であり手動アノテーションされていない、背景優勢の弱教師あり認識問題が生じる。

本論文では、中距離樹種分類のための深度正則化スパース Multiple Instance Learning フレームワークを提案する。画像を局所パッチのバッグに変換し、幹バイアス付きサンプリングによって勾配ベースのスコアを用いて構造的に有用なパッチを選択し、attention-based multiple instance learning によって選択されたインスタンスを集約する。深度誘導拡張は学習時のみに用い、前景の樹木構造を保持しながら背景領域を摂動するため、推論時には RGB 画像のみを必要とする。FruitsPark と UrbanStreetTree における実験では、4 シードプロトコルの下で、画像レベルベースライン、スパース MIL、スパース深度拡張 MIL を比較した。UrbanStreetTree の固定予算バックボーン比較では、ViT DepthAug が 95.10% の accuracy と 94.27% の macro-$F_1$ を達成し、対応する ViT GAP ベースラインをそれぞれ 10.49 ポイント、12.50 ポイント上回った。別途実施した DINOv2 の比率スイープでは、大幅に少ないパッチでもピークに近い性能を維持できることが示された。UrbanStreetTree では、バッグ比率 0.4 が 95.98% の accuracy を達成し、最良結果との差は 0.35 ポイントにとどまり、平均インスタンス数は 367.2 ではなく 163.3 であった。これらの結果は、深度正則化スパース MIL が、背景優勢の中距離撮像条件における樹木分類に対して有効かつパッチ効率の高い戦略であることを示している。

## キーワード

Attention-based multiple instance learning、背景バイアス、深度誘導拡張、スパースパッチ選択、樹種分類、都市林、Vision Transformer。

## 1. はじめに

自動樹種分類は、都市緑地管理、生物多様性調査、果樹園モニタリング、スマートシティの環境インベントリを支援する。モバイルおよび地上レベルの撮像は、低コストで導入しやすいため魅力的である。しかし、成功している植物・樹木認識パイプラインの多くは、葉、花、果実、樹皮の近接観察に依存してきた [leafsnap, plantnet, barknet, centralbark]。この仮定は、フィールド調査では常に成り立つわけではない。ユーザは数メートル離れた位置から樹木全体を撮影する場合があり、都市インベントリシステムは歩行者または車両の視点から街路樹を観測する場合がある。このような中距離画像では、樹木は部分的にしか見えず、背景がフレームの大部分を占めることが多い。

本研究は、この撮像条件によって生じる特有の困難に焦点を当てる。樹種認識における識別信号は、幹、枝、樹皮パターン、局所的な樹木構造に集中しているが、これらの領域にはアノテーションが付与されていない。同時に、画像には建物、道路、空、車両、標識、地面のテクスチャ、隣接する植生など、大量のシーン文脈が含まれる。現代の分類器は、このような文脈をショートカットとして利用できる。背景依存性は一般物体認識 [backgroundbias] や植物病害データセット [plantvillagebias] で定量化されており、ラベルと相関する背景アーティファクトだけでも非自明な分類が可能であることが示されている。中距離樹木画像では、物体証拠が疎で背景領域が広いため、このリスクはさらに大きくなる。

近接樹皮分類は、有用だが不完全な先行例である。BarkNet 1.0 と CentralBark は、樹皮が樹木認識のための年間を通じて有効な強い手掛かりであることを示している [barknet, centralbark]。しかし、これらのデータセットは近距離の樹皮画像が中心である。幹を遠距離から観測すると、樹皮テクスチャの実効解像度は低下し、画像上で同じ幹幅に見えても実際の物理スケールは大きく異なる場合がある。したがって、中距離樹種分類には、手動の幹クロップに依存せずに局所証拠を保持し、弱い局所手掛かりをより広い構造情報と組み合わせる方法が必要である。

図 1: 異なる距離で撮影された樹木画像の例。中距離観測では、近接観測よりも背景領域が大きく、局所的な樹皮テクスチャが弱くなる。

本研究では、この設定を背景優勢の Multiple Instance Learning (MIL) 問題として定式化する。画像はパッチのバッグとして表現され、利用できるのは画像レベルの樹種ラベルのみであり、分類器はどのインスタンスが有用かを推定しなければならない。Attention-based MIL (ABMIL) は、インスタンスに重みを割り当て、パッチレベルの診断も可能にする微分可能なプーリング機構を提供する [abmil]。しかし、高解像度画像を密に処理する場合、全パッチ MIL は計算コストが高い。また近年のスパース MIL 研究では、インスタンス選択、マスキング、枝刈りは単なる実装上の詳細ではなく、重要な設計軸であることが示されている [pamil, mhim, acmil, keypatches, hdmil]。単純なスパースサンプリングは、有用な幹または枝のパッチを破棄する可能性がある。そこで本研究では、スライディングウィンドウによるカバレッジと勾配ベースの構造スコアを組み合わせて候補パッチの疎な部分集合を選択する、幹バイアス付きサンプリング (trunk-biased sampling; TBS) を導入する。さらに、学習時の背景ショートカットを低減するため、深度由来の前景構造を保持しながら背景領域を摂動する深度誘導拡張 (DepthAug) を用いる。深度はデータ構築または学習時拡張にのみ用いられ、最終モデルは推論時に RGB 画像のみを使用する。

本論文の貢献は以下の通りである。

- 地上レベルの中距離樹種分類を、背景優勢の弱教師ありパッチ集約問題として定式化する。
- 幹バイアス付きパッチ選択、Vision Transformer ベースの特徴抽出、ABMIL 集約を組み合わせたスパース MIL フレームワークを提案する。
- 推論時に深度を必要とせず、背景ショートカットを抑制する学習時のみの正則化として、深度誘導拡張を組み込む。
- FruitsPark と UrbanStreetTree において、統一された 4 シードプロトコルの下で、画像レベルベースライン、スパース MIL、スパース DepthAug MIL を評価する。
- accuracy とパッチ予算のパレートフロンティアを定量化し、特に UrbanStreetTree において、大幅に少ないインスタンスでピークに近い accuracy を維持できることを示す。

## 2. 関連研究

### 2.1 樹種分類

樹種認識は、葉、花、果実、樹皮、樹木全体、航空画像、衛星画像、LiDAR を用いて研究されてきた。LeafSnap と Pl@ntNet は、近接した器官観察に基づく画像ベースの植物識別を普及させた [leafsnap, plantnet]。樹皮ベースの認識は、樹皮を年間を通して観測できるため魅力的である。BarkNet 1.0 は大規模な樹皮データセットを導入し、CNN ベース分類器による強い性能を報告した [barknet]。CentralBark は、深層学習ベースの樹種認識のために標準化された樹皮画像をさらに拡張した [centralbark]。これらの研究は樹皮テクスチャの価値を示しているが、幹が小さく背景干渉が大きい本研究の中距離設定とは、近距離という前提が異なる。

都市樹木データセットも拡大している。UrbanStreetTree は、中国の都市で収集された街路樹器官の画像分類とインスタンスセグメンテーションのための包括的な公開データセットであり、枝を含み、50 種にわたる 41,467 枚の高解像度分類画像を含む [urbanstreettree]。StreetTree のような近年の大規模ベンチマークも、街路レベルの樹種分類が活発な研究分野になりつつあることを示している [streettree]。データセット規模の貢献とは対照的に、本研究は、対象構造が疎に分布し、幹レベルのアノテーションが利用できない場合に、中距離 RGB 画像をどのように分類するかを扱う。

### 2.2 背景バイアスと中距離認識

背景バイアスは、視覚認識における主要な信頼性問題である。Xiao らは、背景のみの情報でも ImageNet-9 の非自明な分類を支えられること、また敵対的に選ばれた背景が、前景が正しく分類される場合でも誤った予測を誘発し得ることを示した [backgroundbias]。Noyan は、PlantVillage の背景画素にラベルと相関するアーティファクトが含まれ、少数の背景画素だけを用いる分類器でもランダム推測を大きく上回ることを報告した [plantvillagebias]。背景画素が画像を占有することが多い中距離樹木認識では、これらの知見は直接関係する。

中距離撮像はまた、局所テクスチャの識別価値を低下させる。CNN はテクスチャ手掛かりに強く依存することが知られており、形状バイアスを高めた学習はロバスト性を改善できる [texturebias]。樹種分類において、これは近接樹皮テクスチャだけでは不十分であり、モデルはシーン文脈ショートカットを避けながら、局所テクスチャの残存情報と幹・枝構造を組み合わせる必要があることを意味する。

### 2.3 MIL、Transformer、弱い局所証拠

MIL は、サンプルをインスタンスのバッグとして表現し、バッグレベルのラベルのみを用いる [dietterich, carbonneau]。Deep MIL はこの考え方を画像分類と自動アノテーションに拡張し [deepmil]、ABMIL はインスタンス集約のための学習可能な attention プーリングを導入した [abmil]。弱教師あり全スライド画像解析では、CLAM が attention とクラスタリング制約を用いて、スライドレベルラベルから代表的な高価値領域を同定する [clam]。DSMIL は、デュアルストリーム構造によりインスタンスレベルとバッグレベルの証拠をモデル化する [dsmil]。TransMIL のような Transformer ベースの MIL 手法は、全スライド画像におけるインスタンス間相関をさらにモデル化する [transmil]。これらの研究は、弱教師から識別的な局所領域を推定するという点で概念的に関連している。

本タスクは、有用な証拠の分布が異なる。病理画像では、多くの組織パッチがタスク関連の形態を含む可能性がある。一方、中距離樹木画像では、有用な幹・枝パッチは少数派になり得るのに対し、視覚的に強い背景パターンは豊富である。そのため、本研究では ABMIL と明示的なスパースパッチ選択、および深度正則化学習を組み合わせる。

Vision Transformer (ViT) は、画像をパッチトークンとして処理し、長距離関係をモデル化できる [vit]。Swin Transformer は、階層的な shifted-window attention を導入した [swin]。DINO、DINOv2、DINOv3 のような自己教師あり視覚バックボーンは、大規模な非ラベルコーパスから汎用的な視覚表現を学習する [dino, dinov2, dinov3]。本研究では、画像レベル設定と MIL 設定の両方でこれらのバックボーンを評価し、背景優勢のスパース集約下で有効かどうかに注目する。

### 2.4 スパースかつ効率的なインスタンス選択

近年の MIL 文献では、抽出されたすべてのインスタンスを集約器で処理すべきかという点がますます問われている。PAMIL は、全スライド分類のために動的ポリシー駆動のインスタンスサンプリングを導入し、サンプリングを学習問題の一部として明示的に扱う [pamil]。MHIM-MIL は、顕著で容易なインスタンスをマスクすることで、より困難だが有用なインスタンスの発見を促す [mhim]。ACMIL は attention の集中を分析し、ごく少数のインスタンスが attention を支配し得ることを示し、過度な集中を抑制する機構を動機づける [acmil]。Key Patches Are All You Need は、一部の領域だけを用いてもドメイン内診断性能を保ち、分布シフトへのロバスト性を改善できることを報告している [keypatches]。HDMIL は、無関係なパッチを除去することにより、ギガピクセル画像の高速かつ高精度な分類を目指す [hdmil]。

これらの研究の多くは計算病理分野のものであり、樹種認識ではない。それでも、弱ラベル、多インスタンス、高冗長性という同じ状況を扱うため、方法論的には重要である。本研究の貢献は、スパース MIL を中距離樹木画像へ移し、学習済みの病理特化 relevance モジュールではなく、応用固有の幹事前知識と学習時のみの深度拡張を用いる点にある。

### 2.5 深度誘導学習

MiDaS や Depth Anything V2 のような単眼深度推定モデルは、RGB 画像からシーン構造を推定する [midas, depthanythingv2]。深度は推論時モダリティとしてだけでなく、学習時の構造信号としても利用できる。例えば DepthG は、教師なしセグメンテーションのために深度誘導の特徴相関とサンプリングを用い、推論時に必ずしも深度を必要としない [depthg]。本研究の DepthAug 戦略は、この学習時のみの原則に従う。深度由来の前景マスクは学習時の背景摂動に用いられるが、実運用時の分類器は RGB のみで動作する。

## 3. 手法

### 3.1 概要

提案フレームワークは、中距離樹木画像を構造的に有用なパッチのスパースバッグに変換し、視覚バックボーンでパッチ特徴を抽出し、ABMIL で集約する。背景文脈への依存を低減するため、深度誘導拡張を学習時に適用する。図 2 は全体の処理を示す。

図 2: スパース MIL フレームワークの概要。現在の主系列は、推論時の RGB パッチ、幹バイアス付きスパースサンプリング、ABMIL 集約、学習時のみの深度誘導拡張を用いる。

### 3.2 画像からバッグへの構成

入力画像 $x$ が与えられたとき、候補パッチはスライディングウィンドウによって生成される。候補パッチ集合を $\mathcal{C}(x)=\{q_1,\ldots,q_M\}$ とする。スパースバッグ $\mathcal{B}(x)$ は、バッグ比率 $\rho \in (0,1]$ に従って $\mathcal{C}(x)$ から選択され、おおよそ $K=\lfloor \rho M \rfloor$ 個のパッチが MIL モデルで処理される。固定パッチ数とは異なり、比率ベースの定義は、有効解像度が異なる画像間でも利用可能な局所証拠の同程度の割合を保持する。

### 3.3 幹バイアス付きサンプリング

TBS は、樹皮エッジ、幹境界、枝輪郭のような高周波の樹木構造を含む可能性が高い候補パッチを優先する。各候補パッチ $q$ について、Sobel スコアを次のように計算する。

```math
S_{\mathrm{sobel}}(q)=\frac{1}{|q|}\sum_{u \in q}\|\nabla I(u)\|_2,
```

ここで、$I$ は画像強度、$u$ はパッチ内の画素を表す。候補パッチは $S_{\mathrm{sobel}}$ によって順位付けされる。幹バイアスパラメータ $\beta$ は、サンプラが高スコアパッチをどれだけ強く優先するかを制御する。論文向け設定では、スライディング候補、$\beta=0.5$、候補倍率 4.0、ストライド 160 画素を用いる。

この選択は意図的に単純かつ決定的である。幹を正確にセグメントすることを目的としていない。むしろ、スパースバッグを構造的に有用な領域へ寄せつつ、ABMIL がどのパッチが有用かを学習するために十分な多様性を保持することを目的とする。

### 3.4 パッチ特徴抽出と ABMIL

選択された各パッチ $p_i \in \mathcal{B}(x)$ は、バックボーン特徴抽出器 $\phi(\cdot)$ によって符号化される。

```math
z_i=\phi(p_i)\in\mathbb{R}^{d}, \qquad i=1,\ldots,K.
```

ABMIL の attention スコアは次の通りである。

```math
e_i=w^{\top}\tanh(Wz_i+b),
```

正規化された attention 重みは次の通りである。

```math
a_i=\frac{\exp(e_i)}{\sum_{j=1}^{K}\exp(e_j)}.
```

バッグ表現は次のように計算される。

```math
s=\sum_{i=1}^{K} a_i z_i,
```

クラス確率は、線形分類器と softmax によって生成される。

```math
p(y=c\mid x)=\mathrm{softmax}(W_c s+b_c).
```

学習では、attention 分布に対する balanced elastic-net 正則化を加えた cross-entropy を最小化する。

```math
\mathcal{L}=\mathcal{L}_{\mathrm{CE}}+\lambda_1\|a\|_1+\lambda_2\|a\|_2^2.
```

主プロトコルでは、$\lambda_1=\lambda_2=5\times 10^{-4}$ とし、entropy 正則化は無効化する。

### 3.5 深度誘導拡張

DepthAug は、RGB 画像から推定された疑似深度マップを用いて、前景の樹木構造を保持し背景領域を摂動した学習画像を構成する。その目的は、深度を推論時モダリティとして追加することではなく、学習中にラベルと相関する背景ショートカットを弱めることである。

$D(x)$ を正規化された相対深度マップ、$m(x)$ を $D(x)$ のしきい値処理とモルフォロジカル平滑化によって得られる前景マスクとする。深度拡張された学習画像は次のように形成される。

```math
\tilde{x}=m(x)\odot x + (1-m(x))\odot T(x,r),
```

ここで $T(\cdot)$ は、ぼかし、色変換、圧縮、外部自然シーン $r$ との置換などの背景摂動を適用する。検証画像とテスト画像にはこの拡張を適用しない。推論時には、モデルは RGB 画像のみを受け取り、同じ TBS と ABMIL パイプラインを使用する。

### 4.2 比較モデル

全体比較には 3 つのグループを含める。第一に、画像レベル GAP ベースラインは、ResNet18 [resnet]、ViT [vit]、DINO/DINOv2/DINOv3 ViT バックボーン [dino, dinov2, dinov3]、Swin Transformer [swin] を用いる。これらのベースラインでは、バックボーンは学習可能である。第二に、スパース MIL モデルは、凍結バックボーンと $\rho=0.3$ の TBS を用いる。第三に、スパース DepthAug モデルは同じスパース MIL 設定を用いるが、`only_blur/aug_midas_large` 変種から構成された深度拡張データセットで学習する。報告する全結果は、0042、2024、2025、3407 の 4 シードを用いる。

パレート効率実験では、バックボーンを DINOv2、モデルファミリを DepthAug 付き TBS スパース MIL に固定し、4 シードで $\rho \in \{0.1,0.2,\ldots,1.0\}$ をスイープする。accuracy、macro-$F_1$、バッグあたり平均インスタンス数を報告する。パッチインスタンス数はエンコーダ負荷の直接的な代理指標である。デプロイメント固有の時間主張を行う前には、壁時計時間のレイテンシを別途測定すべきである。

さらに、2 つのスパース選択チェックを含める。第一に、固定予算条件のアブレーションでは、全学習/全評価、全学習/スパース評価、ランダムスパース学習/評価、TBS スパース学習/評価、TBS スパース学習 + DepthAug 後のスパース RGB 評価を比較する。第二に、比率別比較では、同じバッグ比率において、提案する TBS-DepthAug 構成とランダムスパースサンプリングを評価する。これらの分析は、提案するサンプリングと拡張戦略が単なるパッチ数削減を上回るかどうかを判断するために用いる。

### 4.3 実装詳細

すべてのモデルは 30 epoch 学習する。GAP ベースラインでは batch size 32 を用い、MIL モデルでは各画像がパッチバッグとして表現されるため batch size 1 を用いる。データセット固有の正規化は training split から計算する。スパース MIL と DepthAug モデルは、softmax ABMIL attention、Sobel TBS スコアリング、$\lambda_1=\lambda_2=5\times10^{-4}$、entropy 正則化なし、自動混合精度を用いる。実験は、NVIDIA RTX A6000 GPU、CUDA 12.9、AMD Ryzen Threadripper 7960X CPU、62 GiB のシステムメモリを備えたワークステーションで実行した。

### 5.2 深度誘導拡張の効果

DepthAug は UrbanStreetTree で最も有効である。対応するスパース MIL モデルに対するペアードシードの macro-$F_1$ 改善は、ViT で +3.74 ポイント、DINOv2 で +3.59、DINOv3 で +2.57、Swin で +4.28 である。これは、背景ショートカットが深刻な街路シーンにおいて、学習時の背景摂動が有用であるという解釈を支持する。

FruitsPark では効果が混在する。DINOv2 と ViT は DepthAug から改善を得るが、DINOv1 と DINOv3 は改善しない。この違いは解釈上重要である。FruitsPark は特定の公園環境で収集された validation-only データセットであり、強い画像レベル foundation model は安定したドメイン規則性から利益を得る可能性がある。したがって、FruitsPark は手法の優越性主張の唯一の根拠ではなく、手法挙動の二次的検証として用いる。

### 5.3 スパース選択アブレーション

図 7 の固定予算条件アブレーションでは、スパース推論だけで十分か、また TBS と DepthAug がランダムスパース学習を上回るかを比較する。FruitsPark では、このプロトコルにおいて全学習/全評価が 88.6% accuracy に達する。全学習からスパース評価へ切り替えると accuracy は 78.9% に低下し、評価予算を削減するだけでは十分ではないことが示される。ランダムスパース学習/評価は 76.3%、TBS スパース学習/評価は 78.2%、TBS スパース + DepthAug は 85.0% に改善する。UrbanStreetTree では、全学習/全評価が 94.2% に達し、全学習/スパース評価、ランダムスパース、TBS スパース、TBS スパース + DepthAug はそれぞれ 90.4%、88.3%、89.7%、91.3% である。したがって、固定されたスパース予算では、TBS はランダムスパースサンプリングに対して小さな利得を与え、DepthAug は最大の追加利得をもたらす。全パッチ処理は依然として強い参照点であるが、スパース選択と学習時背景摂動を組み合わせることで、その差は大幅に縮小される。

### 5.5 定性的 attention 挙動

Attention 可視化は、高重みパッチが幹、枝、背景領域のどれに対応するかを確認する診断として有用である。しかし、attention 重みはそれ自体を因果的説明として扱うべきではない [attentionnotexplanation]。そのため、本研究では attention 挙動を補助的な診断証拠としてのみ用い、主要な主張は 4 シードの定量比較とパレート分析に基づける。選択パッチの可視化は、定量実験で用いた正確な TBS-DepthAug 設定から生成された場合に最も有益である。したがって、以前の内部変種からの可視化は主要証拠として使用しない。

## 6. 考察

### 6.1 なぜスパース MIL が有効か

中距離樹木分類では、モデルは細かな局所証拠を保持しながら、対象全体の構造を推論する必要がある。標準的な画像レベルのリサイズは、特に樹木が小さい領域を占める場合に、幹や樹皮情報を希釈する可能性がある。ABMIL は画像全体を 1 つのグローバル記述子に押し込むことを避け、モデルが局所パッチへ重みを割り当てられるようにする。TBS は、特徴抽出前にインスタンス数を削減し、バッグを構造的パッチへバイアスすることで、これを実用的にする。ランダム比較実験は、ランダムスパースサンプリングが弱いベースラインではないことも示している。パッチ数削減だけでも、いくつかの比率では競争力を持つ。したがって、TBS-DepthAug の価値は、すべてのパッチ予算でランダムサンプリングを支配することではなく、固定スパース条件を改善し、実用的なスパース運用点を保ちながら、より高い最良 accuracy 点を生み出すことにある。

### 6.2 なぜ DepthAug は UrbanStreetTree で有効か

UrbanStreetTree は複雑な街路背景と独立した test split を含み、背景ショートカット抑制を評価する強い設定である。バックボーンをまたいで一貫した DepthAug の利得は、学習時に深度由来の背景領域を摂動することで、分類器がシーン文脈のみに依存することを防ぐという解釈を示唆する。重要なのは、DepthAug が推論時の深度センサ要求を課さない点であり、これはモバイルおよび低コスト展開に不可欠である。

### 6.3 データセット依存の挙動

FruitsPark は異なるパターンを示す。DINOv3 GAP が全体で最良の手法であり、DepthAug はすべてのバックボーンを改善するわけではない。これは提案フレームワークを否定するものではない。むしろ、スパース深度正則化 MIL の効果が、データセット構造、split 設計、背景変動に依存することを示している。FruitsPark には独立した test set がないため、validation 性能は収集ドメインの規則性を反映している可能性がある。今後の研究では、汎化性能をより厳密に測定するため、場所または季節の hold-out split を追加すべきである。

### 6.4 効率性の解釈

パレート結果は、本手法の実用的価値の中心である。UrbanStreetTree では、候補パッチの 40% を用いるだけで、最良設定から 0.35 ポイント以内の accuracy を維持しながら、平均インスタンス数を 367.2 から 163.3 へ削減できる。ランダムスパースベースラインも同程度の範囲に到達するが、その最良点は提案手法よりわずかに低い。これは、スパースパッチ処理自体が重要であり、TBS と DepthAug が高性能領域で追加利得を提供することを示している。ただし、これはまだ完全なレイテンシ主張ではない。前処理、パッチスコアリング、GPU バッチング、特徴抽出オーバーヘッドを含む end-to-end の測定が必要である。それでも、平均インスタンス数はエンコーダ負荷の直接的かつ再現可能な代理指標であり、観測されたフロンティアは、ピークに近い accuracy のために全パッチ処理が不要であることを示している。

### 6.5 限界

本研究には 4 つの主要な限界がある。第一に、FruitsPark には独立した test set がない。第二に、現在の TBS スコアは手設計の Sobel 指標であり、学習可能またはクラス認識的なパッチ選択によりロバスト性が改善する可能性がある。第三に、疑似深度マップは、密な植生や特殊な照明条件で失敗する可能性がある。第四に、本論文はパッチインスタンス効率を報告しており、完全なデプロイメントレイテンシベンチマークではない。今後の実験では、hold-out location、end-to-end timing、confidence-aware depth augmentation、近年の学習型 efficient MIL selector との直接比較を含めるべきである。

## 7. 結論

本論文では、背景優勢条件下における中距離樹種分類のための深度正則化スパース MIL フレームワークを提示した。本手法は、各画像を構造的に有用なパッチのスパースバッグとして表現し、ABMIL でパッチ証拠を集約し、学習時に深度誘導拡張を用いて背景ショートカットを低減しながら、RGB のみの推論を維持する。固定予算 4 シード実験では UrbanStreetTree で強い利得が示され、ViT DepthAug は 95.10% accuracy と 94.27% macro-$F_1$ を達成した。固定予算アブレーションでは、TBS と DepthAug がランダムスパースサンプリングに対してスパース学習を改善することが示された。一方、比率別比較では、ランダムスパースサンプリングが依然として強いベースラインであることも示された。パレート分析は、特に UrbanStreetTree において、より少ないパッチインスタンスでピークに近い性能を維持できることを示した。バッグ比率 0.4 は、使用インスタンス数を半分未満にしながら、最良結果から 0.35 ポイント以内にとどまった。これらの知見は、複雑な実世界シーンにおける地上レベル樹木認識の実用的方向性として、スパースで深度正則化された弱教師あり集約を支持するものである。
